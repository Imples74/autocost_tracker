# Техническое задание на проект: AutoCost Tracker

## Тема проекта

**Веб-сервис учета и анализа расходов на автомобиль для частных автовладельцев с целью прогнозирования ежемесячных затрат**

---

## 1. Название и концепция проекта

**AutoCost Tracker** — это веб-сервис для учета, анализа и прогнозирования расходов на содержание автомобиля.

Проект предназначен для частных автовладельцев, которые хотят понимать, сколько денег уходит на эксплуатацию автомобиля, какие категории расходов являются самыми затратными и какую сумму примерно нужно заложить на следующий месяц.

Сервис позволяет пользователю добавлять один или несколько автомобилей, фиксировать расходы по категориям, просматривать статистику и получать прогноз ежемесячных затрат на основе накопленных данных.

Главная идея проекта заключается не только в хранении расходов, но и в их аналитической обработке. Пользователь получает не просто таблицу расходов, а наглядную финансовую картину по каждому автомобилю.

---

## 2. Цель проекта

Цель проекта — разработать веб-сервис на Django, который позволяет частным автовладельцам:

- вести учет расходов на один или несколько автомобилей;
- распределять расходы по категориям;
- анализировать структуру затрат;
- просматривать статистику по периодам;
- строить графики расходов;
- прогнозировать примерные расходы на следующий месяц.

---

## 3. Проблема, которую решает проект

Многие автовладельцы не ведут системный учет расходов на автомобиль. Из-за этого сложно понять:

- сколько реально стоит содержание машины;
- какая категория расходов забирает больше всего денег;
- как меняются траты от месяца к месяцу;
- сколько денег нужно заранее подготовить на следующий месяц.

AutoCost Tracker решает эту проблему за счет хранения расходов, их группировки, анализа и визуализации.

---

## 4. Целевая аудитория

Основная целевая аудитория проекта — частные автовладельцы.

К ним относятся:

- владельцы одного личного автомобиля;
- пользователи, у которых есть несколько автомобилей;
- люди, которые хотят контролировать расходы на топливо, ремонт, обслуживание, страховку, налоги и другие статьи затрат;
- пользователи, которым важно планировать личный бюджет.

---

## 5. Роли пользователей

### 5.1. Гость

Гость может:

- открыть главную страницу сайта;
- ознакомиться с описанием сервиса;
- перейти на страницу регистрации;
- перейти на страницу входа.

Гость не может:

- добавлять автомобили;
- создавать расходы;
- просматривать аналитику;
- видеть данные других пользователей.

### 5.2. Авторизованный пользователь

Авторизованный пользователь может:

- зарегистрироваться и войти в личный кабинет;
- добавить один или несколько автомобилей;
- редактировать и удалять свои автомобили;
- добавлять записи о расходах;
- просматривать список своих расходов;
- фильтровать расходы по автомобилю, категории и дате;
- просматривать аналитику по выбранному автомобилю;
- видеть графики расходов;
- получать прогноз расходов на следующий месяц.

Пользователь имеет доступ только к своим автомобилям и своим расходам.

### 5.3. Администратор

Администратор использует стандартную административную панель Django.

Администратор может:

- просматривать пользователей;
- управлять автомобилями;
- управлять категориями расходов;
- просматривать и редактировать расходы;
- просматривать сохраненные прогнозы.

## 6. Модели данных

В проекте планируется использовать 4 кастомные модели:

1. `Car`
2. `ExpenseCategory`
3. `Expense`
4. `MonthlyForecast`

Также будет использоваться стандартная модель пользователя Django — `User`.

---

## 6.1. Модель Car

Модель `Car` хранит информацию об автомобиле пользователя.

### Назначение

Эта модель нужна для того, чтобы пользователь мог вести учет расходов отдельно по каждому автомобилю.

### Основные поля

| Поле | Описание |
|---|---|
| `user` | Владелец автомобиля |
| `brand` | Марка автомобиля |
| `model` | Модель автомобиля |
| `year` | Год выпуска |
| `fuel_type` | Тип топлива |
| `engine_volume` | Объем двигателя |
| `mileage` | Текущий пробег |
| `created_at` | Дата добавления автомобиля |

---

## 6.2. Модель ExpenseCategory

Модель `ExpenseCategory` хранит категории расходов.

### Назначение

Эта модель нужна для группировки расходов. Благодаря категориям пользователь может видеть, на что именно уходит больше всего денег.

### Основные поля

| Поле | Описание |
|---|---|
| `name` | Название категории |
| `description` | Описание категории |
| `is_regular` | Является ли расход регулярным |

### Примеры категорий

- Топливо
- Ремонт
- Техническое обслуживание
- Страховка
- Налоги
- Мойка
- Парковка
- Штрафы
- Прочее

---

## 6.3. Модель Expense

Модель `Expense` хранит конкретную запись о расходе.

### Назначение

Это основная модель проекта. В ней фиксируются все траты пользователя по конкретному автомобилю.

### Основные поля

| Поле | Описание |
|---|---|
| `car` | Автомобиль, к которому относится расход |
| `category` | Категория расхода |
| `amount` | Сумма расхода |
| `expense_date` | Дата расхода |
| `mileage` | Пробег на момент расхода |
| `description` | Комментарий пользователя |
| `created_at` | Дата создания записи |

---

## 6.4. Модель MonthlyForecast

Модель `MonthlyForecast` хранит результат прогноза расходов на месяц.

### Назначение

Эта модель нужна для сохранения рассчитанных прогнозов. Прогноз формируется на основе истории расходов по выбранному автомобилю.

### Основные поля

| Поле | Описание |
|---|---|
| `car` | Автомобиль, для которого сделан прогноз |
| `forecast_month` | Месяц прогноза |
| `predicted_amount` | Прогнозируемая сумма расходов |
| `calculation_description` | Описание метода расчета |
| `created_at` | Дата создания прогноза |

---

## 7. Связи между моделями

В проекте используются следующие связи:

### User → Car

Один пользователь может иметь несколько автомобилей.

### Car → Expense

Один автомобиль может иметь много записей о расходах.

### ExpenseCategory → Expense

Одна категория может использоваться во многих расходах.

### Car → MonthlyForecast

Для одного автомобиля может быть создано несколько прогнозов.

## 10. Аналитический модуль

Для выполнения требования по технической сложности в проекте будет реализован аналитический модуль.

Аналитика будет строиться на основе данных из модели `Expense`.

Планируется использовать:

- Django ORM для фильтрации и агрегации данных;
- Pandas для обработки расходов и расчета статистики;
- Chart.js для отображения графиков на страницах сайта.

---

## 10.1. Расчеты

В проекте будут реализованы следующие расчеты:

### 1. Общая сумма расходов

Система суммирует все расходы пользователя по выбранному автомобилю за выбранный период.

### 2. Расходы по категориям

Система группирует расходы по категориям и считает сумму по каждой категории.

### 3. Средний месячный расход

Система группирует расходы по месяцам и рассчитывает среднее значение.

### 4. Самая затратная категория

Система определяет категорию, по которой сумма расходов максимальна.

### 5. Прогноз на следующий месяц
